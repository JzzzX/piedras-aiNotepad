# AI Notepad 前端界面改版方案

这是一份为您量身定制的前端界面改版方案，融合了 Granola 的高级质感、中文排版的呼吸感，以及即刻/小宇宙式的轻盈克制。

## 1）整体改版思路

**设计策略：去线框化、通过底色分层、强化内容呼吸感。**

目前的四栏布局容易显得像“功能堆叠的管理后台”，我们要将其转化为一个“安静的数字书写与记录空间”。整体界面将采用一种极微的对比度策略：用带有极浅暖灰或冷灰的底层背景（如 `bg-gray-50/50` 或 `bg-[#F9F9FB]`）作为画布，而承载核心内容的转写区和笔记区则使用纯白（`bg-white`）并辅以大圆角和极其克制的微阴影或透明度边框。

**为什么这样改？**
Granola 的高级感来源于它的“拟物但扁平”——它像一张放在整洁桌面上的纸。对于中文用户而言，高密度的汉字容易带来视觉疲劳，因此我们需要比英文 SaaS 更大的行高（`leading-relaxed` 或 `1.6`~`1.8`）、更明确的字重区分（标题用中黑/SemiBold，正文用常规/Regular），以及更多的留白。去掉生硬的分割线，改用背景色阶差来区分侧边栏和主工作区，能立刻消除系统的“机器味”，增加工具的“陪伴感”和“手感”。

## 2）分区域改版建议

### 左侧会议列表 (Sidebar)
*   **当前问题：** 列表容易显得枯燥，选中态和未选中态区分生硬，时间等辅助信息喧宾夺主。
*   **改版目标：** 营造“私人手账目录”的感觉，弱化边界，强调当前聚焦的记录。
*   **视觉建议：** 移除侧边栏右侧的实体分割线（`border-r`）。整个侧边栏背景与全局背景融合。列表项（Item）默认无背景、无边框；Hover 时给出极浅的底色（`hover:bg-black/5`）；选中态（Active）使用纯白底色（`bg-white`）+ 极轻微阴影（`shadow-sm`）+ 微小的圆角（`rounded-xl`），让选中的记录“浮”出来。
*   **交互建议：** 标题文字超长时使用优雅的渐变截断或 `line-clamp-2`，避免直接截断带来的破裂感。

### 顶部工具栏 (Header/Topbar)
*   **当前问题：** 标题输入框像表单，主按钮突兀，整体缺乏统一的头部气场。
*   **改版目标：** 沉浸式的文档标题体验，克制的操作指引。
*   **视觉建议：** 标题输入框去除所有边框和底色（`border-none bg-transparent`），字号加大（如 `text-2xl` 或 `text-3xl`），字重加粗，让它看起来就像是一篇文档的标题，而不是一个 `<input>`。状态指示（如“正在录音”）使用柔和的呼吸灯动效（Pulse）配合胶囊形状（Pill）。主按钮（如“开始录音”）取消大面积高亮实心，改为带有品牌色的幽灵按钮或带有一点磨砂玻璃质感（Backdrop Blur）的悬浮按钮。
*   **交互建议：** 往下滚动内容时，顶部标题栏可以加上一层极轻的毛玻璃效果（`backdrop-blur-md bg-white/70`）并吸顶。

### 实时转写区 (Transcript Panel)
*   **当前问题：** 像聊天记录，说话人标签视觉干扰大，空状态冷冰冰。
*   **改版目标：** 清晰易读的对话流，像剧本一样的排版。
*   **视觉建议：** 背景使用纯白。说话人标签（Speaker）不要用鲜艳的色块，改用小字号、全大写（如果有英文）、中性色（`text-gray-400`），并带有轻微的字间距（`tracking-widest`）。转写文本字号适中（`text-[15px]` 或 `text-base`），行高拉大。
*   **空状态：** 放置一个精致的线框 Icon 或温和的提示语，例如“保持安静，准备聆听...”，字色极浅，居中放置，带一点缓慢的入场动画。

### 笔记编辑区 (Note Editor)
*   **当前问题：** 像普通的 TextArea，缺乏“写作区”的神圣感。
*   **改版目标：** 提供沉浸、无干扰的 Markdown 写作体验。
*   **视觉建议：** 这部分必须是整个屏幕视觉最纯净的地方。纯白背景，取消周围的边框，仅仅通过大面积的留白（`p-8` 或 `p-12`）来界定输入区域。如果需要区分转写和笔记，可以在两者之间留出稍微大一点的 `gap`。

### 右侧 AI 助手区 (AI Copilot/Assistant)
*   **当前问题：** 像传统的智能客服对话框，割裂感强。
*   **改版目标：** 像是一个贴在主笔记旁边的“灵感便利贴”或“智能侧边栏”。
*   **视觉建议：** 背景色使用非常淡的品牌色或暖灰色（例如 `bg-indigo-50/30` 或 `bg-stone-50/50`），以此与纯白的转写/笔记区区分开。气泡框（Bubble）不要有尖角，全部使用完全圆角（`rounded-2xl`）。AI 的回复不需要头像，直接通过字体颜色或气泡底色区分。

## 3）视觉设计规范 (Tailwind 优先)

*   **页面背景层级：**
    *   底层背景（Base）：`bg-[#F9F9FB]` 或 `bg-gray-50`（如果是暗黑模式准备，用 `bg-neutral-50` 更温和）。
    *   内容层（Surface）：`bg-white`。
    *   悬浮/选中层（Elevated）：`bg-white shadow-[0_2px_8px_rgba(0,0,0,0.04)]`。
*   **字号与行高（中文优化）：**
    *   大标题（Title）：`text-2xl md:text-3xl font-semibold leading-tight text-gray-900`
    *   副标题/说话人：`text-xs font-medium text-gray-400 tracking-wider`
    *   正文（Body）：`text-[15px] md:text-base text-gray-700 leading-[1.75]`
    *   辅助文本（Caption）：`text-sm text-gray-500`
*   **间距系统 (Spacing)：**
    *   栏间距（Gaps）：放弃 1px 边框，使用 `gap-4` 或 `gap-6` 的空隙。
    *   卡片内边距（Padding）：尽量宽松，`p-6` 或 `p-8` 起步。
*   **圆角建议 (Border Radius)：**
    *   外层大容器：`rounded-3xl`
    *   内层卡片/面板：`rounded-2xl`
    *   按钮/标签：`rounded-full` 或 `rounded-xl`
*   **边框使用原则：**
    *   **极度克制**。能用背景色区分就不加边框；必须加时使用 `border border-black/5` 或 `border-gray-100`。
*   **色彩系统：**
    *   主文本：`#111827` (gray-900)
    *   次文本：`#4B5563` (gray-600)
    *   主品牌色：推荐类似小宇宙的克制亮色，或极光质感的深色，例如靛蓝色 `text-indigo-600 bg-indigo-50`。
    *   危险/录音状态：`text-rose-500 bg-rose-50`

## 4) 适合前端落地的实现建议

*   去除主要区块的 `border` 和 `border-gray-200`，替换为 `bg-gray-50/50` 页面底层加各区域的 `bg-white rounded-2xl shadow-sm`。
*   增加 `gap-4` 或 `gap-6` 以取代区块紧贴的情况。
*   左侧列表项从边框切换为 hover/active 底色反馈。
*   输入框从 `border input` 改为 `border-transparent hover:bg-gray-50 focus:bg-white`。
